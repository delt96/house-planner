# 부착물(문·창·콘센트) 도면 직접 배치 & 방 자석 스냅 설계 문서

## 1. 배경 / 문제

현재 문·창·콘센트는 사이드 패널의 방 카드에서 폼으로만 추가한다: 벽을 북/동/남/서
드롭다운으로 고르고 "모서리에서 몇 cm"를 숫자로 입력해야 한다. 도면을 보면서
"이 벽이 북쪽인가, 기준 모서리가 어느 쪽인가"를 머릿속으로 변환해야 해서 배치가
불편하다.

또한 방들이 서로를 모르는 독립 사각형이라, 여러 방을 이어 붙여 실제 집 평면도를
만들 때 벽끼리 딱 맞추기 어렵고 전체가 "따로 노는" 느낌이 난다.

오늘의집(직접 조작 + 실시간 거리 표시)과 CAD(스탬프식 클릭 배치, 수치 보정)를
참고해 도면 위 직접 배치로 전환한다. 이 앱의 입력값은 줄자 실측값이므로,
**마우스로 대략 놓고 숫자로 정확히 보정**하는 흐름을 표준으로 한다.

## 2. 범위

- 부착물 **생성**: 도구바 클릭 → 배치 모드 → 도면 벽 클릭으로 생성
- 부착물 **이동**: 도면에서 기호를 벽 따라 드래그 (같은 방 안에서 벽 전환 포함)
- 부착물 **수정**: 기호 클릭 → 떠 있는 속성 카드에서 수치·열림방향 보정, 삭제
- 방 **자석 스냅**: 방 드래그 시 다른 방 벽·모서리에 착 붙는 스냅 + 가이드라인
- 패널 방 카드 정리: 추가/수정 폼 제거, 목록은 유지

### 비범위 (이번엔 안 함)
- 맞붙은 벽의 문을 "방↔방 연결 문"으로 인식하는 데이터 모델 확장
- 방 겹침 방지(충돌 밀어내기)
- 부착물의 다른 방으로 이동 (삭제 후 재생성으로 대체)
- 터치(모바일) 지원 — 기존과 동일하게 마우스 전용
- 같은 벽 안 부착물 겹침 방지 (창 아래 콘센트처럼 현실에 존재하므로 허용)

## 3. 상호작용 설계

### 3.1 생성 — 클릭 배치 모드

- 캔버스 상단 범례 영역에 도구바: `[🚪 문] [⊞ 창문] [⚡ 콘센트]` 토글 버튼.
- 버튼 클릭 → 배치 모드 진입. 버튼 활성 표시, 캔버스 커서 crosshair,
  힌트 문구 "벽을 클릭해 배치 · ESC 취소" 표시. 같은 버튼 재클릭 또는 ESC로 종료.
- 배치 모드 중 도면에서 마우스를 움직이면:
  - 모든 방의 벽 중 커서에서 가장 가까운 벽을 찾는다 (임계거리 30cm 이내,
    캔버스 좌표 기준. 벗어나면 유령 숨김).
  - 그 벽 위 투영 지점에 반투명 유령 기호를 그린다 (기존 도면 기호와 동일 렌더러).
  - 유령 양쪽으로 모서리까지 거리를 실시간 표시한다 (`←130→ ░░░ ←210→`).
  - offset은 5cm 단위 스냅, `[0, 벽길이−폭]`으로 클램프.
- 클릭 → `POST /rooms/:id/features`로 기본 치수 생성:
  - 문: 폭 80 × 통과높이 204, 열림 `in-left`
  - 창문: 폭 150 × 창높이 120, 턱 90
  - 콘센트: 바닥에서 30
- 생성 직후 해당 기호가 **자동 선택되어 속성 카드가 열린다** — 실측값을 바로
  보정할 수 있다. 배치 모드는 유지되므로 다시 벽을 클릭하면 카드가 닫히고
  다음 것이 연속 생성된다 (콘센트 여러 개 등).
- 유령의 폭이 벽 길이를 초과하면 유령을 빨간색으로 표시하고 클릭을 무시한다.
- 방이 하나도 없으면 힌트 문구를 "방을 먼저 추가하세요"로 바꾼다.
- 배치 모드 중에는 방·가구 드래그를 비활성화한다 (모드 우선).

### 3.2 이동 — 벽 따라 드래그

- 기호 mousedown 후 드래그: 커서에서 가장 가까운 벽(같은 방의 4개 벽만)에
  재투영하며 벽을 따라 미끄러진다. 드래그 중 양쪽 모서리 거리 라벨을 실시간 표시.
- 이동 거리 3px 미만의 mouseup은 드래그가 아니라 클릭(선택)으로 처리한다.
- 드롭 시 5cm 스냅·클램프 후 `PATCH /features/:id` (wall, offset_cm) 저장.
- 기호의 mousedown은 방 드래그로 전파되지 않게 막는다.

### 3.3 수정 — 떠 있는 속성 카드

- 기호 클릭 → 선택 → 기호의 화면 좌표 근처에 HTML 오버레이 속성 카드 표시
  (캔버스 wrap `position:relative` + absolute div). 기존 텍스트 칩은 카드로 대체.
- 카드 구성:
  - 헤더: 종류 아이콘·라벨 + 삭제 버튼
  - 위치: "왼쪽 모서리에서 [  ]cm / 오른쪽 모서리에서 [  ]cm" **양방향 입력**
    — 한쪽을 고치면 반대쪽은 자동 계산 (`벽길이 − 폭 − 입력값`). 실측 시 어느 쪽
    모서리에서 쟀든 그대로 입력 가능.
  - 종류별 필드: 문 = 폭·통과높이·열림방향, 창 = 폭·창높이·턱높이, 콘센트 = 바닥높이
  - 열림방향은 드롭다운 대신 **안/밖 × 좌/우 토글 버튼 2쌍** — 누르는 즉시 저장되어
    도면 스윙 호가 갱신된다.
- 수치 입력은 blur 또는 Enter에서 `PATCH /features/:id` 저장. 실패 시 카드에
  오류 메시지 표시, 값 롤백.
- 카드 밖 클릭 또는 ESC → 선택 해제·카드 닫힘. ESC는 카드가 열려 있으면 카드를
  먼저 닫고, 다음 ESC에서 배치 모드를 종료한다.
- 패널 방 카드의 부착물 목록 항목 클릭 시에도 같은 선택 상태가 되어 카드가 열린다.

### 3.4 패널(방 카드) 정리

- 부착물 목록은 유지 (요약 확인용). "수정" 버튼은 제거하고 항목 클릭 = 도면 선택.
- `＋ 문/창문/콘센트` 추가 폼과 인라인 수정 폼(`FeatureForm`)은 제거 — 도구바와
  속성 카드가 대체한다. 항목별 삭제 버튼과 천장 높이 입력은 유지.

### 3.5 방 자석 스냅

- 방 드래그 중, 제안 위치의 4개 변을 다른 방들의 변과 비교한다:
  - 마주보는 변끼리 (내 왼쪽 벽 ↔ 상대 오른쪽 벽 등) 거리 **15cm 이내**면 착 붙음.
  - 붙는 축이 정해지면 직교 방향으로도 상대 방 모서리와 15cm 이내일 때 모서리 정렬.
  - 수평·수직 각각 독립적으로 스냅 (동시에 둘 다 걸릴 수 있음).
- 스냅이 걸린 벽을 따라 **파란 가이드라인**을 드래그 중에만 표시한다.
- 스냅이 안 걸리면 기존 10cm 그리드 스냅 그대로.
- 드롭 시 스냅 반영된 좌표로 `PATCH /rooms/:id` 저장.

## 4. 기술 설계

**API·DB 변경 없음.** 기존 `POST /rooms/:id/features`, `PATCH /features/:id`,
`DELETE /features/:id`, `PATCH /rooms/:id`와 서버 검증(벽 앵커·벽 길이 체크)을
그대로 사용한다.

### geometry.js에 추가할 순수 함수 (단위 테스트 대상)

```
nearestWallPoint(rooms, cmX, cmY, thresholdCm)
  → { roomId, wall, offsetCm } | null      커서 좌표를 가장 가까운 벽에 투영
clampFeatureOffset(room, wall, offsetCm, widthCm)
  → offsetCm                               [0, 벽길이−폭] 클램프
snapRoomPosition(room, otherRooms, proposedX, proposedY, thresholdCm)
  → { x, y, guides: [{axis, positionCm, fromCm, toCm}] }   자석 스냅 + 가이드라인
```

마우스 좌표 → 캔버스 cm 좌표 변환은 `svg.getBoundingClientRect()` 기준
(viewBox가 픽셀과 1:1이므로 rect 원점 차감 후 `pxToCm`).

### 컴포넌트 구성

- `FeatureToolbar.jsx` — 도구바 (배치 모드 토글). LayoutPage가 모드 state 소유.
- `FeaturePropertyCard.jsx` — 오버레이 속성 카드. props: feature, room, 저장/삭제
  콜백. 자체적으로 폼 상태 관리.
- `FeatureSymbols.jsx` — 유령(ghost) 렌더링 재사용을 위해 단일 기호 렌더러
  `FeatureSymbol` 분리. 칩 렌더링은 속성 카드로 대체되므로 제거.
- 거리 라벨(양쪽 모서리 거리)은 SVG `<text>` + 보조선으로, 생성 유령과 이동
  드래그에서 공용.
- `RoomCard.jsx` — `FeatureForm`·추가 버튼 제거, 목록 클릭 = 선택.
- `LayoutPage.jsx` — state 추가: `placementMode`(null|'door'|'window'|'outlet'),
  `ghost`(투영 결과), 기호 드래그 state, 방 드래그에 스냅 적용.

## 5. 에러 처리

- 서버 400(검증 실패): 속성 카드에 메시지 표시, 입력값 롤백. 생성 클릭 실패 시
  캔버스 상단 기존 오류 영역에 표시.
- 배치 모드에서 유효하지 않은 위치(벽 초과·벽 없음)는 클라이언트에서 선차단
  (빨간 유령·클릭 무시)하되, 서버 검증은 최후 방어선으로 유지.

## 6. 테스트 전략

- **geometry 단위 테스트:** `nearestWallPoint`(각 벽·임계 밖·여러 방),
  `clampFeatureOffset`(경계), `snapRoomPosition`(변 스냅·모서리 정렬·비스냅).
- **컴포넌트 테스트 (기존 vitest + testing-library 패턴):**
  - 도구바 클릭 → 모드 진입 → 캔버스 mousemove → 유령 렌더 → 클릭 →
    `api.createFeature` 호출 및 자동 선택 확인
  - 기호 드래그 → `api.updateFeature`에 스냅된 offset 전달 확인
  - 속성 카드: 오른쪽 모서리 기준 입력 → offset 환산 저장, 열림 토글 → 즉시 저장
  - 방 드래그 스냅: 임계 이내 드롭 → 스냅된 좌표로 `api.updateRoom` 호출
- 기존 `RoomCard` 폼 관련 테스트는 목록/삭제/천장 테스트로 대체.

## 7. 구현 순서 (요약)

1. geometry 순수 함수 3종 + 단위 테스트
2. 도구바 + 배치 모드 + 유령/거리 라벨 + 클릭 생성
3. 속성 카드 (선택·수정·삭제, 양방향 위치 입력) + RoomCard 정리
4. 기호 벽 따라 드래그
5. 방 자석 스냅 + 가이드라인
